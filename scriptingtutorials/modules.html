Roblox Studio Modules, what are they and how could we use them?

In this tutorial we will be explaining everything about modules and how you can use them.

What are module scripts?

Module scripts are global scripts that return one value when called. In order to receive the value, you must call the require() function on it. Module scripts can contain lots of data that could be used across multiple scripts. Now let's move on to the next question, which is how you could use them.

How could you use them?

They are mostly used as global variables and global functions; you should only use them when you want to access something across multiple scripts.

How to use them?

Using them is pretty simple. You can create a module anywhere and access it from a script. You should store your modules in Server Storage for server scripts and in Replicated Storage for local scripts.

Code example:

Module script inside Server Storage:

                    
local ModuleMessage = "Password: 12345"      

return ModuleMessage

Server script inside Server Script Service:

                    
local ModuleMessage = require(game.ServerScriptService.ModuleScript) 

print(ModuleMessage)

Downsides?

Modules basically have no downsides. They are used for almost everything.
